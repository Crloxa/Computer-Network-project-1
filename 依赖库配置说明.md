# VisualNet-QR-Video 依赖库配置说明

## 项目依赖概述

本项目需要以下两个核心依赖库才能正常编译和运行：

1. **OpenCV 4.1.2** - 图像处理和视频处理库
2. **FFmpeg** - 视频编解码库

由于这些库文件较大，无法直接上传到GitHub，需要用户自行下载并配置。

## 依赖库下载和配置步骤

### 第一步：下载依赖库

#### 1. OpenCV 4.1.2 下载

**下载地址（选择其中一个）：**
- 官方下载：https://opencv.org/releases/
- GitHub Releases：https://github.com/opencv/opencv/releases/tag/4.1.2

**下载文件：**
- Windows版本：`opencv-4.1.2-vc14_vc15.exe`
- 或者直接下载预编译包

#### 2. FFmpeg 下载

**下载地址：**
- 官方下载：https://ffmpeg.org/download.html
- Windows预编译版：https://www.gyan.dev/ffmpeg/builds/

**下载文件：**
- Windows版本：`ffmpeg-release-full.7z` 或类似版本

### 第二步：解压和目录结构

#### 项目根目录结构要求：
```
Viusal_Net/
├── newvisualnet/          # 源代码目录
├── opencv/                 # OpenCV库目录（需要手动创建）
│   └── build/
│       ├── include/        # 头文件
│       ├── x64/
│       │   └── vc16/
│       │       └── lib/   # 库文件
│       └── bin/           # DLL文件
├── ffmpeg/                # FFmpeg目录（需要手动创建）
│   └── bin/
│       └── ffmpeg.exe     # FFmpeg可执行文件
└── x64/
    └── Debug/             # 编译输出目录
```

#### 具体操作步骤：

1. **创建依赖库目录**
   ```bash
   # 在项目根目录下创建以下目录
   mkdir opencv
   mkdir ffmpeg
   ```

2. **配置OpenCV**
   - 运行下载的OpenCV安装程序，选择解压到 `opencv` 目录
   - 或者手动将OpenCV文件复制到对应目录
   - **最终目录结构：**
     ```
     opencv/
     └── build/
         ├── include/opencv2/          # 头文件
         ├── x64/vc16/lib/             # .lib库文件
         └── bin/                      # .dll动态库文件
     ```

3. **配置FFmpeg**
   - 解压FFmpeg到 `ffmpeg` 目录
   - **最终目录结构：**
     ```
     ffmpeg/
     └── bin/
         ├── ffmpeg.exe
         ├── ffprobe.exe
         └── 其他FFmpeg相关文件
     ```

### 第三步：验证依赖库配置

#### 检查OpenCV配置：
```bash
# 检查头文件是否存在
ls opencv/build/include/opencv2/opencv.hpp

# 检查库文件
ls opencv/build/x64/vc16/lib/opencv_world4120.lib

# 检查DLL文件
ls opencv/build/bin/opencv_videoio_ffmpeg4120_64.dll
```

#### 检查FFmpeg配置：
```bash
# 检查FFmpeg可执行文件
ls ffmpeg/bin/ffmpeg.exe

# 验证FFmpeg版本
ffmpeg/bin/ffmpeg.exe -version
```

## 项目配置说明

### Visual Studio 项目配置

项目已经配置好依赖路径，具体配置在 `newvisualnet.vcxproj` 文件中：

#### 包含目录（Include Directories）：
```xml
<AdditionalIncludeDirectories>$(SolutionDir)opencv\build\include;%(AdditionalIncludeDirectories)</AdditionalIncludeDirectories>
```

#### 库目录（Library Directories）：
```xml
<AdditionalLibraryDirectories>$(SolutionDir)opencv\build\x64\vc16\lib;%(AdditionalLibraryDirectories)</AdditionalLibraryDirectories>
```

#### 依赖库（Additional Dependencies）：
- Debug模式：`opencv_world4120d.lib`
- Release模式：`opencv_world4120.lib`

### 运行时依赖

#### 必需的DLL文件：
编译完成后，需要将以下DLL文件复制到可执行文件同目录：

1. **OpenCV DLLs：**
   - `opencv_world4120.dll` (Release)
   - `opencv_world4120d.dll` (Debug)
   - `opencv_videoio_ffmpeg4120_64.dll`

2. **FFmpeg DLLs：**
   - FFmpeg相关的DLL文件（如果使用动态链接）

#### 自动复制DLL脚本（可选）：
可以创建一个批处理文件来自动复制DLL：

```batch
@echo off
REM 复制OpenCV DLL到输出目录
copy "opencv\build\bin\opencv_world4120.dll" "x64\Debug\"
copy "opencv\build\bin\opencv_world4120d.dll" "x64\Debug\"
copy "opencv\build\bin\opencv_videoio_ffmpeg4120_64.dll" "x64\Debug\"

REM 复制FFmpeg可执行文件
copy "ffmpeg\bin\ffmpeg.exe" "x64\Debug\"

echo DLL文件复制完成！
pause
```

## 常见问题解决

### 1. 编译错误：找不到OpenCV头文件
**解决方法：**
- 检查 `opencv/build/include` 目录是否存在
- 确认OpenCV版本为4.1.2
- 检查项目配置中的路径是否正确

### 2. 链接错误：找不到OpenCV库
**解决方法：**
- 检查 `opencv/build/x64/vc16/lib` 目录
- 确认库文件名是否正确（opencv_world4120.lib）
- 检查Visual Studio平台工具集版本

### 3. 运行时错误：缺少DLL
**解决方法：**
- 将OpenCV的DLL文件复制到可执行文件目录
- 或者将OpenCV的bin目录添加到系统PATH环境变量

### 4. FFmpeg相关错误
**解决方法：**
- 确认 `ffmpeg/bin/ffmpeg.exe` 文件存在
- 检查FFmpeg版本兼容性
- 确保FFmpeg可执行文件有执行权限

## 依赖库版本兼容性

- **OpenCV**: 4.1.2（推荐），其他4.x版本可能兼容
- **FFmpeg**: 最新稳定版即可
- **Visual Studio**: 2019或更高版本
- **平台**: x64 Windows

## 替代方案（如果不想手动配置）

### 使用vcpkg包管理器：
```bash
# 安装vcpkg
git clone https://github.com/Microsoft/vcpkg.git
cd vcpkg
./bootstrap-vcpkg.bat

# 安装依赖
./vcpkg install opencv4[ffmpeg]:x64-windows
./vcpkg install ffmpeg:x64-windows

# 集成到Visual Studio
./vcpkg integrate install
```

### 使用Conan包管理器：
```bash
# 安装Conan
pip install conan

# 配置依赖
conan install . --build missing
```
