# ISO QR 四色增强承载方案（可优化选项）

> 历史参考警告：本文档是旧 ISO 路线上的可选增强方向，不是当前默认主线。
>
> 当前默认执行链已经切回 `docs/protocols/protocol_v1.md` 的 `V1.6-108-4F`；不要把本文档当成当前可执行方案。

## 1. 目标与定位

- 在不破坏标准 ISO QR 本体的前提下，通过 4 色副载层提升单帧净吞吐。
- 保留现有黑白链路作为稳定兜底，确保在彩色链路退化时仍可解码。
- 将彩色增强限定为可切换选项，避免影响当前主线交付与联调。

## 2. 总体思路

- 基础层：中心保持标准黑白 ISO QR，继续承载关键 header 与基础 payload。
- 增强层：在 carrier 可用区域新增规则四色网格，承载增量 payload。
- 组合策略：解码时先走基础层，再合并增强层；增强层失败时自动降级到基础层。

## 3. 四色符号设计

### 3.1 色表与符号映射

- 每个色块承载 `2 bits`。
- 建议映射（可按实测调整）：
  - `00 -> Red`
  - `01 -> Green`
  - `10 -> Blue`
  - `11 -> Yellow`
- 建议使用高饱和、色距大的目标颜色，并在解码端使用每帧颜色标定进行自适应校正。

### 3.2 副载层放置范围

- 不覆盖标准 QR 本体及 quiet zone。
- 不覆盖四角 marker 的几何定位区域。
- 优先在 carrier 的边框区域或外围固定数据带上布置彩色网格。

## 4. 编码侧流程（新增）

1. 使用现有流程生成黑白 ISO QR 基础层。
2. 将增量 payload 做 FEC 编码与交织。
3. 把增量 bitstream 按 `2 bits` 映射为四色符号。
4. 按固定网格位置渲染彩色 cell。
5. 输出与基础层一致的帧序号关联信息，保证可重组。

## 5. 解码侧流程（新增）

1. 继续使用现有 marker 检测和透视拉正。
2. 在拉正图上进行颜色校正：白平衡归一、亮度归一、颜色矩阵校正。
3. 对彩色网格按 cell 中心窗口采样，输出颜色类别与置信度。
4. 将颜色类别反映射为 bitstream，执行反交织与 FEC 纠错。
5. 与基础层 payload 按帧号合并，完成最终重组。

## 6. 组内分工增量

### 6.1 二维码矫正组需要新增

- 新增颜色标定 patch 设计与检测。
- 在透视拉正后增加颜色校正流程（建议 `BGR -> Lab/HSV` 双通道实验）。
- 输出校正质量指标：标定色块偏差、各色混淆矩阵、置信度热图。

### 6.2 编码组需要新增

- 新增四色符号映射与渲染模块。
- 新增增强层 header（palette_id、grid 规格、FEC 参数、增强层 CRC）。
- 增加基础层/增强层的分层封包与对齐策略。

### 6.3 解码组需要新增

- 新增四色分类器（自适应阈值或最近颜色中心分类）。
- 新增软判决接口，向 FEC 提供置信度。
- 新增失败降级策略：增强层失败不影响基础层输出。

## 7. 推荐起步参数

- 调研阶段仅启用 `4 色 + 强 FEC`，先追求稳定再追求峰值吞吐。
- 先做录屏链路验证，再做实拍链路验证。
- 维持现有 `iso133 + Q/H + 较大 canvas` 作为几何与采样密度基础。

## 8. 风险与约束

- 风险来源：自动白平衡、曝光波动、视频压缩、屏摄反光。
- 约束结论：增强层必须可关闭，且不得破坏当前 ISO 主线兼容性。
- 工程要求：彩色方案独立开关、独立统计、独立报告。

## 9. 验收建议

- 成功率：增强层成功重组率、基础层兜底成功率。
- 吞吐：相同 `profile/ecc/canvas` 下的净有效吞吐提升比。
- 质量：各颜色误判率、FEC 前后误码率、端到端重组正确率。

## 10. 与主线关系

- 本方案是可优化选项，不替代当前主线。
- 当前默认交付、演示和验收仍基于黑白 ISO QR 主线。
- 若后续进入实现阶段，建议新增独立协议分支并保持文档隔离。
