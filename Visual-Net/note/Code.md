第五版编码测试（按手绘稿重制排布）

输出分辨率：1330*1330  
实际分辨率：133*133  
最外圈：安全区 * 2（二维码外侧安全区）  
识别码：左上、右上、左下各一个 21*21 主定位块  
额外识别码：右下角预留一个 21*21 区域，保留 7*7 辅助定位块。该辅助定位块相对右下角向左、向上各偏移 1 个像素，其余非数据区全部留白  

Header 区域：位于左上定位块右侧，范围为 `row 3..20, col 21..76`，总大小 18*56。  
Header 内部仍只承载 3 组语义不变的字段：  

1. 帧标志 + 尾长  
2. 校验码  
3. 帧号  

为了提高稳定性，Header 按 3 个高度为 6 的子带分布，每一位在横向使用 3 列重复绘制。字段语义、位序和原实现保持一致，不新增协议字段。  

数据区共 4 块，按从上到下、从左到右的区域顺序写入：  

1. 数据区1：`row 3..20, col 77..108`  
2. 数据区2：`row 21..108, col 3..129`  
3. 数据区4：`row 109..111, col 3..129`  
4. 数据区3：`row 112..129, col 21..111`  

顶部 `col 109..111` 为纯白保留区。  
右下 21*21 保留区内，除 7*7 辅助定位块外，其余非数据区全部保持纯白。  

本阶段编码仍然使用黑白二值，不使用彩色三通道写入。  
所有数据区采用“行优先单元流”写入，不再要求每一行都单独满足 8bit 对齐。只要整个区域累计单元数足够，就按连续 bit 流写入。  

当前阶段协议有效载荷仍保持 1242 字节，不扩容。  
如果数据区总容量超过 1242 字节，则剩余单元保持随机黑白乱码。  

注意：

1. 这次改动只调整编码端排布。  
2. 解码端仍未适配新版 133*133 排布。  
3. 当前数据区2与数据区4使用 `col 3..129`，顶部和两侧收边后的空白保持为非数据区。
