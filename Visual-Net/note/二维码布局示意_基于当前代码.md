- 逻辑尺寸：`108 x 108`

- 每个逻辑点放大：`10 x 10 px`
- 最外圈安全区：四边各 `2` 个点
- 大定位器：左上、右上、左下各 `18 x 18`
- 小定位器：右下额外定位器约 `9 x 9`

## 2. 定位器在哪里

### 2.1 三个大定位器

- 左上定位器：`x=0..17, y=0..17`
- 右上定位器：`x=90..107, y=0..17`
- 左下定位器：`x=0..17, y=90..107`

它们的作用：

- 用于检测二维码位置
- 用于透视矫正
- 用于旋转方向判断

这 3 个 `18x18` 块不只是纯黑白方块，而是把“识别图案 + 外围留白”一起打包进去了。
`Code.md` 里给出的口径是 `2:2:6:2:2`，对应这一版大定位器的层次宽度。

### 2.2 右下小定位器

- 中心约在：`(102, 102)`
- 实际绘制范围约：`x=98..106, y=98..106`

它的作用：

- 给右下角增加一个额外定位基准
- 提高透视矫正时对右下角的约束

## 3. 帧头信息在哪里

这一版没有单独开一个大矩形 header 区，而是把“帧头”拆成左上定位器下方的 3 行：

- 第 1 行：帧标志 `y=18, x=2..17`
- 第 2 行：校验码 `y=19, x=2..17`
- 第 3 行：帧号 `y=20, x=2..17`

### 3.1 帧标志区

位置：

- `y=18, x=2..17`
- 共 `16` 个黑白点

存放内容：

- 前 `4` 点：帧类型
- 后 `12` 点：尾长 `tailLen`

当前代码里的帧类型颜色模式：

- `WWBB`：起始帧 `Start`
- `BBWW`：终止帧 `End`
- `WWWW`：单帧起止合一 `StartAndEnd`
- `BBBB`：普通中间帧 `Normal`

说明：

- `W` 表示白点
- `B` 表示黑点
- 后 `12` 位记录当前帧尾长，按小端位序写入
- 如果不是结束帧，编码端会强制把 `tailLen` 当作满帧 `1242`

### 3.2 校验码区

位置：

- `y=19, x=2..17`
- 共 `16` 个黑白点

存放内容：

- `checkcode`，`uint16_t`

算法：

1. 将 payload 按 2 字节一组做异或
2. 如果 payload 是奇数长度，最后 1 字节按高 8 位参与异或
3. 再异或 `len`
4. 再异或 `frameBase`
5. 再异或帧类型标志：`(isStart << 1) + isEnd`

### 3.3 帧号区

位置：

- `y=20, x=2..17`
- 共 `16` 个黑白点

存放内容：

- `FrameNo % 65536`

作用：

- 作为帧序号
- 解码时用于检测重帧、跳帧

## 4. 数据区在哪里

当前代码把有效数据拆成 7 个矩形区，按固定顺序依次写入。

### 4.1 区域清单

| 区域 | 左上角 `(y,x)` | 尺寸 `(高x宽)` | 容量 |
|---|---:|---:|---:|
| 左侧区 Area0 | `(21, 2)` | `69 x 16` | `138 B` |
| 上侧区 Area1 | `(2, 18)` | `16 x 72` | `144 B` |
| 中央区 Area2 | `(18, 18)` | `72 x 72` | `648 B` |
| 右侧区 Area3 | `(18, 90)` | `72 x 16` | `144 B` |
| 下侧区 Area4 | `(90, 18)` | `16 x 72` | `144 B` |
| 右下区A Area5 | `(90, 90)` | `8 x 16` | `16 B` |
| 右下区B Area6 | `(98, 90)` | `8 x 8` | `8 B` |

总容量：

- `138 + 144 + 648 + 144 + 144 + 16 + 8 = 1242 bytes`

### 4.2 写入顺序

写入顺序不是简单整图 row-major，而是按下面顺序拼接：

1. Area0 左侧区
2. Area1 上侧区
3. Area2 中央区
4. Area3 右侧区
5. Area4 下侧区
6. Area5 右下区A
7. Area6 右下区B

也就是说：

- 先填左边长条
- 再填上边横条
- 再填中间大块
- 最后填右边、下边和右下角小块

## 5. 每个数据区里怎么存

### 5.1 颜色和比特含义

当前代码实际使用黑白二值：

- 白色 = `0`
- 黑色 = `1`

### 5.2 单字节写法

每连续 `8` 个点，表示 `1` 个字节。

写入方式：

- 从左到右写
- 按小端位序写入
- 也就是第一个点对应 bit0，第二个点对应 bit1，依次到 bit7

### 5.3 区域内扫描方式

每个矩形区内部都是：

- 先按行
- 每行从左到右
- 每 `8` 列组成 `1` 字节

所以一个区域的读取顺序是：

1. 第 1 行左侧 8 点
2. 第 1 行右侧 8 点（如果宽度是 16）
3. 第 2 行左侧 8 点
4. 第 2 行右侧 8 点
5. 依此类推

## 6. 一张单帧码图里实际存的内容

如果把一帧看成“帧头 + 数据体”，那么它存的是：

### 6.1 定位相关

- 左上大定位器
- 右上大定位器
- 左下大定位器
- 右下小定位器
- 四边安全区

这些都不是业务数据，而是为了识别、裁剪、矫正。

### 6.2 控制信息

- 帧类型
- 该帧有效尾长
- 校验码
- 帧号

### 6.3 业务数据

- 当前帧 payload，最多 `1242 bytes`
- 如果最后一帧不足 `1242 bytes`，后面会补随机字节
- 真正有效长度由帧标志中的 `tailLen` 决定

---

## 7. 你做自己的码制时最值得借鉴的点

### 7.1 布局思路

- 三角大定位器负责主定位
- 右下额外小定位器负责补右下角约束
- 帧头放在固定小区域，便于优先解析
- 数据区拆成多个矩形，绕开定位器和边缘保留区

### 7.2 工程思路

- 先定位，再读帧头，再读数据区
- 用帧号解决重帧和跳帧
- 用轻量异或校验先做快速筛错
- 最后一帧用尾长裁剪真实数据
