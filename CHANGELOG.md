# Changelog

## 2026-03-13
- 将 `Visual-Net` 编码端二维码几何从 `108x108` 调整为 `133x133`，输出分辨率同步调整为 `1330x1330`。
- 三个主定位块调整为 `21x21`，右下角改为 `21x21` 预留区，保留原 7x7 辅助定位块。
- 维持当前每帧 `1242` 字节有效载荷不变，仅完成编码端迁移；解码端适配留待后续阶段完成。
- 按手绘稿重制 Header 与 4 个数据区排布，Header 改到左上定位块右侧，数据区切换为新的顶带、中区、过渡带、底带布局。
- Header 行范围进一步调整为 `row 3..20`，并将右下 `21x21` 保留区除 `7x7` 辅助定位块外的所有非数据区统一改为空白。
- 右下 `7x7` 辅助定位块整体向左、向上各平移 1 个像素。
- 新增 `Layout_DEBUG` 调试预览开关，可额外输出带彩色分区边界的 `_layout` 预览图，不影响正式黑白编码帧。
- 数据区 `1/2/4/3` 在编码逻辑上合并为一个连续数据流，Header 宽度收窄为 `48` 列并取消左右留白，顶部数据区1扩展到 `row 3..20, col 69..111`。
- 右下 `21x21` 预留区中，除辅助定位块、其外侧 2 行 / 2 列安全区和最外缘 `row/col 130..132` 外，其余 L 形单元并入数据流。
- 合并后总数据单元数为 `14212`，末尾固定保留 `4` 个 padding 单元，得到 `14208` 个可写单元，可按 8bit 对齐。
- Header 编码改回 `1bit = 1 个逻辑像素` 的 `3x16` 点阵写法，顶部释放空间继续回填到数据区1，新的总数据单元数为 `15028`，其中 `15024` 个可写单元可按 8bit 对齐。
- 当前 `133x133` 协议的每帧有效载荷上限提升到 `1878` 字节；本阶段仅调整编码端，解码端适配留待后续完成。
